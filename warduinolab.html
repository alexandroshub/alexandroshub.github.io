The Platform

What is Arduino?

Arduino is a low-cost, open-source microcontroller platform designed for building electronics, automation, and robotics projects. Created in 2005 at the Interaction Design Institute in Ivrea, Italy, it was named after Arduin — a military commander of Ivrea in the 11th century.

At its core, an Arduino board holds a microcontroller — a small computer on a chip that can read inputs (sensors, buttons, signals) and turn them into outputs (motors, LEDs, displays). You program it using Wiring C, a simplified version of C/C++ that runs inside the Arduino IDE.

What makes it powerful is the combination of low cost, a huge community, and the ability to go from a blinking LED to a fully autonomous vehicle using the same platform. That's exactly what I did with the Michatron AGV project.

The software IDE is based on the Processing environment — open source, cross-platform, and backed by a library ecosystem that covers everything from Wi-Fi modules to servo motors to machine learning at the edge.

Tutorial

Getting Started with Arduino

Five steps that take you from zero to running your first program on hardware. No prior experience required.

1

Download and install the Arduino IDE

Go to arduino.cc → Software → Downloads. Choose your OS (Windows, macOS, Linux). The IDE includes everything you need: editor, compiler, serial monitor, and board manager. First time installing drivers on Windows? The installer handles it automatically.

Arduino IDE 2.x USB Driver Board Manager
2

Understand the two essential functions

Every Arduino sketch has exactly two mandatory functions. setup() runs once at power-on — use it to configure pins and initialize libraries. loop() runs forever in a continuous cycle — this is where your logic lives.

void setup() { pinMode(LED_BUILTIN, OUTPUT); // configure pin 13 as output } void loop() { digitalWrite(LED_BUILTIN, HIGH); // LED on delay(1000); // wait 1 second digitalWrite(LED_BUILTIN, LOW); // LED off delay(1000); // wait 1 second }
setup() loop() pinMode() digitalWrite()
3

Read analog inputs — the LDR example

Analog pins (A0–A5) read values from 0 to 1023. A photoresistor (LDR) changes resistance based on light — perfect for building an automatic light switch. Connect the LDR and a fixed resistor as a voltage divider to A0, then read it.

int ldrPin = A0; int ledPin = 9; int threshold = 500; // adjust for your lighting void setup() { pinMode(ledPin, OUTPUT); Serial.begin(9600); } void loop() { int ldrValue = analogRead(ldrPin); Serial.println(ldrValue); // monitor in IDE serial console if (ldrValue < threshold) { digitalWrite(ledPin, HIGH); // dark = LED on } else { digitalWrite(ledPin, LOW); // bright = LED off } delay(100); }
analogRead() LDR Voltage Divider Serial Monitor
4

Control a servo with a potentiometer

A potentiometer gives you an analog input from 0 to 1023. Use map() to convert it to a servo angle (0–180°). This is the foundation for manual control systems, robotic joints, and camera gimbals.

#include <Servo.h> Servo myServo; int potPin = A0; int servoPin = 9; void setup() { myServo.attach(servoPin); } void loop() { int potValue = analogRead(potPin); int angle = map(potValue, 0, 1023, 0, 180); myServo.write(angle); delay(15); // give servo time to move }
Servo.h map() Potentiometer PWM
5

Debug with the Serial Monitor

The Serial Monitor is the Arduino equivalent of console.log. Open it from Tools → Serial Monitor (or Ctrl+Shift+M). Make sure the baud rate matches what you set in Serial.begin(). Use it to print sensor values, debug logic, and understand what your hardware is actually doing in real time.

Common Questions

Arduino FAQ

The questions I get most often — answered directly.

Which Arduino board should I start with?

The Arduino Uno is the right starting point for almost everyone. It has a forgiving 5V logic level, 14 digital I/O pins, 6 analog inputs, and more tutorials than any other board. Once you understand the Uno, moving to a Nano (smaller form factor) or a Mega (more pins) is straightforward. Avoid the ESP32 or Raspberry Pi Pico as a first board — they're more powerful but introduce Wi-Fi, dual-core, and 3.3V logic complexity you don't need yet.

What's the difference between digital and analog pins?

Digital pins read or write only two states: HIGH (5V) or LOW (0V). Think on/off, true/false. Analog pins (A0–A5 on the Uno) use a built-in ADC to read a continuous voltage range and convert it to a number between 0 and 1023. This is how you read sensors like LDRs, potentiometers, temperature sensors, and microphones. Some digital pins also support PWM (marked with ~) — they can simulate analog output by switching on and off very fast, which is how you control servo speed and LED brightness.

Why does my sensor give unstable or noisy readings?

Noise is one of the first real problems you hit in hardware. The most common causes: insufficient grounding (always connect your sensor's GND to Arduino's GND), long floating wires acting as antennas, or the sensor is sensitive to supply voltage fluctuations. Solutions: add a 100nF decoupling capacitor between VCC and GND close to the sensor, use analogRead() multiple times and average the result, or apply a simple moving average in software. For switches, always use a pull-up or pull-down resistor (or use INPUT_PULLUP in pinMode).

Can Arduino run multiple things at the same time?

Arduino has a single-core processor and no operating system, so it can't truly multitask. However, you can simulate concurrent behavior using non-blocking code with millis() instead of delay(). The trick: track the last time each action ran and only execute it when enough time has passed. This is a fundamental Arduino skill that unlocks real embedded systems thinking. The BlinkWithoutDelay example in the IDE is the best starting point.

What's the maximum current I can draw from a digital pin?

Each digital pin can safely source or sink 40mA maximum, with a recommended operating current of 20mA. The total current across all I/O pins combined shouldn't exceed 200mA. This means you can power a single LED directly (with a series resistor), but you should never connect a motor, relay, or solenoid directly to a pin. For those loads, use a transistor (NPN BJT like 2N2222 or MOSFET like IRLZ44N) or a dedicated motor driver IC like the L298N.

How do I connect to the internet from Arduino?

The standard Uno has no built-in networking, so you add a module. The ESP8266 (ESP-01 or NodeMCU) is the most common choice — it communicates with the Uno over serial using AT commands, or you can program it directly as a standalone board. For a cleaner solution, use the Arduino Uno WiFi Rev2 or move to an ESP32-based board entirely. In my ThingSpeak IoT project, I used an ESP8266 module to push DHT11 sensor data to the cloud using the ThingSpeak API over HTTP GET requests.
